Notifications flow In the agent side, mediators are added as listeners to the RunnerListenersManager.
When the mediators catch a fired event, the create a JMX notification and dispatch it.
In the client side, ClientEventsHub is the listener for notifications.
When a notification arrives, the method invokeDispatcher of the notification is invoked,
an instance of AgentClientListenersManager (which implements the JSystemListeners interface) is passed
to the notification each Notification activates the relevant manager method.
